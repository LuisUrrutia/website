La flexibilidad de React significa que hay muchas formas de resolver problemas. Estos son patrones que han probado ser efectivos.

## Custom Hooks para Reutilización de Lógica

Extrae lógica reutilizable en custom hooks.

```tsx
function useLocalStorage<T>(key: string, initialValue: T) {
	const [value, setValue] = useState<T>(() => {
		const stored = localStorage.getItem(key);
		return stored ? JSON.parse(stored) : initialValue;
	});

	useEffect(() => {
		localStorage.setItem(key, JSON.stringify(value));
	}, [key, value]);

	return [value, setValue] as const;
}
```

## Compound Components

Crea APIs flexibles y declarativas con compound components.

```tsx
<Select value={selected} onChange={setSelected}>
	<Select.Trigger>Elige una opción</Select.Trigger>
	<Select.Options>
		<Select.Option value="a">Opción A</Select.Option>
		<Select.Option value="b">Opción B</Select.Option>
	</Select.Options>
</Select>
```

## Render Props para Flexibilidad

Cuando necesitas máxima flexibilidad en cómo se renderizan las cosas.

```tsx
<DataFetcher url="/api/users">
	{({ data, loading, error }) => (loading ? <Spinner /> : <UserList users={data} />)}
</DataFetcher>
```

## Conclusión

El patrón correcto depende de tu caso de uso. Empieza simple y refactoriza cuando sea necesario.
