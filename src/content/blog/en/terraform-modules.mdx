Infrastructure as Code is only valuable if it's maintainable. Modules are the key.

## Module Structure

Keep a consistent structure across all modules.

```
modules/
  vpc/
    main.tf
    variables.tf
    outputs.tf
    README.md
```

## Input Variables

Make modules configurable with sensible defaults.

```hcl
variable "environment" {
  description = "Environment name (dev, staging, prod)"
  type        = string
}

variable "vpc_cidr" {
  description = "CIDR block for the VPC"
  type        = string
  default     = "10.0.0.0/16"
}
```

## Outputs

Expose values that other modules might need.

```hcl
output "vpc_id" {
  description = "The ID of the VPC"
  value       = aws_vpc.main.id
}

output "private_subnet_ids" {
  description = "List of private subnet IDs"
  value       = aws_subnet.private[*].id
}
```

## Conclusion

Good module design pays dividends in reduced duplication and easier maintenance.
