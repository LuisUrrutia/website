React's flexibility means there are many ways to solve problems. Here are patterns that have proven effective.

## Custom Hooks for Logic Reuse

Extract reusable logic into custom hooks.

```tsx
function useLocalStorage<T>(key: string, initialValue: T) {
	const [value, setValue] = useState<T>(() => {
		const stored = localStorage.getItem(key);
		return stored ? JSON.parse(stored) : initialValue;
	});

	useEffect(() => {
		localStorage.setItem(key, JSON.stringify(value));
	}, [key, value]);

	return [value, setValue] as const;
}
```

## Compound Components

Create flexible, declarative APIs with compound components.

```tsx
<Select value={selected} onChange={setSelected}>
	<Select.Trigger>Choose an option</Select.Trigger>
	<Select.Options>
		<Select.Option value="a">Option A</Select.Option>
		<Select.Option value="b">Option B</Select.Option>
	</Select.Options>
</Select>
```

## Render Props for Flexibility

When you need maximum flexibility in how things render.

```tsx
<DataFetcher url="/api/users">
	{({ data, loading, error }) => (loading ? <Spinner /> : <UserList users={data} />)}
</DataFetcher>
```

## Conclusion

The right pattern depends on your use case. Start simple and refactor when needed.
