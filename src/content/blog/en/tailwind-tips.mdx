Tailwind CSS is powerful, but long class lists can become unwieldy. Here's how to keep things clean.

## Use Component Abstraction

Extract repeated patterns into components, not CSS classes.

```astro
---
interface Props {
	class?: string;
}
const { class: className } = Astro.props;
---

<!-- Card.astro -->
<div class:list={["bg-card rounded-xl p-6 shadow-sm", className]}>
	<slot />
</div>
```

## Leverage class:list

Astro's `class:list` directive makes conditional classes cleaner.

```astro
<button
	class:list={[
		"rounded-lg px-4 py-2 font-medium",
		{
			"bg-primary text-white": variant === "primary",
			"border bg-transparent": variant === "outline",
		},
	]}
>
	<slot />
</button>
```

## Design Tokens in CSS

Define semantic colors that map to your design system.

```css
:root {
	--color-background: oklch(98% 0 0);
	--color-foreground: oklch(20% 0 0);
	--color-primary: oklch(55% 0.2 260);
}
```

## Conclusion

Tailwind works best when combined with good component architecture.
